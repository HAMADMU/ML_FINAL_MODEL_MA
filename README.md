# ML_FINAL_MODEL_MA
# ML-Based Dynamic Stability Prediction in Renewable-Dominated Power Systems

## 1. Introduction

This Git repository contains the code that was developed as part of a Third Year 
Individual Project at the University of Manchester, Department of Electrical and 
Electronic Engineering (2025-2026).

The implemented software involves developing a machine learning pipeline for near-
boundary small-signal dynamic stability risk detection in a power system with high 
penetration of renewable energy sources. For this purpose, operational data of the 
IEEE 68-bus (NETS-NYPS) benchmark system are considered. Since all 19,866 operating 
points in the dataset are stable under the classical eigenvalue criterion (RPRMM_0Hz 
≥ 0), the task was formulated as a near-boundary risky points classification problem. 
In this regard, we label those operating points as "risky" whose stability index 
(RPRMM_0Hz) falls into the range [−0.01; +∞). In such a manner, we obtain 211 risky 
operational points, making up 1.06% of the whole dataset.

Two classification models were developed and tested: baseline logistic regression and 
XGBoost classifier with two-stage feature selection pro

## 4. How to Run

### Step 1 — Preprocessing (if starting from raw Excel data)
If you have access to the original Excel dataset, run the preprocessing script first 
to generate the compressed CSV:

```bash
python data_preprocessing.py
```

This will produce `IEEE68bus_ML_ready_risky.csv.gz` in the working directory.

### Step 2 — Baseline logistic regression model
```bash
python baseline_log_reg.py
```

This script will:
- Load the compressed dataset
- Train the logistic regression pipeline
- Print PR-AUC, confusion matrix, and classification report at default threshold (0.5)
- Perform threshold tuning and print results at tuned threshold
- Save the following figures to the working directory:
  - `fig_confusion_05.png` — confusion matrix at threshold 0.5
  - `fig_confusion_tuned.png` — confusion matrix at tuned threshold
  - `fig_pr_curve.png` — precision-recall curve
  - `fig_proba_hist_logy.png` — predicted probability distribution

### Step 3 — XGBoost model
```bash
python xgboost_model_1.0.py
```

This script will:
- Load the compressed dataset
- Apply two-stage feature selection (variance threshold then XGBoost importance ranking)
- Run SMOTE ratio sensitivity analysis across four configurations
- Train the final XGBoost pipeline with the selected 100 features
- Perform threshold tuning and five-fold stratified cross-validation
- Print all metrics and save the following figures:
  - `xgb_fig_feature_importance.png` — top 20 feature importance bar chart
  - `xgb_fig_cm_default.png` — confusion matrix at threshold 0.5
  - `xgb_fig_cm_tuned.png` — confusion matrix at tuned threshold
  - `xgb_fig_pr_curve.png` — precision-recall curve
  - `xgb_fig_proba_hist.png` — predicted probability distribution

Both scripts are self-contained and can be run in any order once the compressed 
dataset is available.

## 5. Technical Details

### Risk label definition
The binary risk label is defined as:
risky = 1  if RPRMM_0Hz ≥ -0.01
risky = 0  otherwise

The threshold of -0.01 corresponds to a time constant of approximately 100 seconds 
(τ = 1/|σ|), indicating very weak damping and low margin to instability.

### Leakage prevention
All stability indicator columns are removed from the feature set before training:
- `RPRMM_0Hz` (the stability margin used to define the label)
- All `DRLDM` frequency band columns

### Class imbalance handling
- SMOTE oversampling is applied during training only, never to the test set
- The `scale_pos_weight` parameter in XGBoost is set to the negative-to-positive 
  class ratio (~93:1) to further compensate for imbalance
- Evaluation uses PR-AUC and risky-class recall rather than accuracy

### Feature selection (XGBoost only)
A two-stage procedure is applied:
1. Variance threshold filter (threshold = 0.01): reduces 950 features to 827
2. XGBoost importance ranking: retains top 100 features for final training

### Threshold tuning
Both models use a threshold tuning procedure that selects the lowest threshold 
satisfying a minimum recall constraint of 0.90 while maximising F1-score, reflecting 
the asymmetric cost of false negatives in stability screening.

### Cross-validation
Five-fold stratified cross-validation is applied to the XGBoost model to provide a 
more reliable estimate of generalisation performance than a single train-test split.

## 6. Known Issues and Future Improvements

- **SHAP local explanation** has not been implemented. Adding SHAP (Shapley Additive 
  Explanations) would provide per-prediction feature attribution, giving operators 
  insight into the physical variables behind each individual risk prediction. This is 
  identified as the primary direction for future work.

- **Single dataset**: the models were trained and validated on operating points from 
  one simulation run of the IEEE 68-bus system. Generalisation to contingency 
  conditions (e.g. line outages, generator trips) or other benchmark systems has not 
  been tested.

- **Risk threshold sensitivity**: the -0.01 threshold used to define risky operating 
  points is based on decay time constant reasoning. Further calibration against 
  operator-defined stability margins would strengthen the practical applicability of 
  the labelling scheme.

- **Model scope**: only logistic regression and XGBoost are evaluated. Extending to 
  Random Forest, LightGBM, or deep learning models such as MLP would allow broader 
  comparison.

- **Multiple stability indices**: the current formulation uses only RPRMM_0Hz. 
  Incorporating multiple frequency band indices from the DRLDM columns could extend 
  the classification to cover inter-area, sub-synchronous, and high-frequency modes.
